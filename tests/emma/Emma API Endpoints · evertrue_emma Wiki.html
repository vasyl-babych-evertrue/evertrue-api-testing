      Get a single Account by ID

      GET /emma/v1/account/{id}?oid={oid}

      {id} is the id of the account you want to return. {oid} is required to look up the account.

      Example Request

      GET /emma/v1/account/1?oid=148

      Example Response Body

    "oid": 148,

      Create an Account

      POST /emma/v1/account/?oid=53

      oid is a required parameter to make sure the account is set up properly.

      Example Request Body

      {
    "remote_id": 124587,
    "name": "John Jacob",
    "public_api_key": "YmxhYmtqaWJ1Cg==",
    "private_api_key": "bXkgbm1hbWUgaXNqZmtqYXdvaWVqZgo="
}

      Example Response Body

      {
    "id": 3,
    "oid": 53,
    "remote_id": 124587,
    "name": "John Jacob",
    "public_api_key": "YmxhYmtqaWJ1Cg==",
    "private_api_key": "bXkgbm1hbWUgaXNqZmtqYXdvaWVqZgo=",
    "created_at": 1542048293008,
    "updated_at": null,
    "invalidated_at": null

      Update an Account

      PATCH /emma/v1/account/{id}

      {id} is the id of the Account that is being updated. This replaces the values specified in the body as keys with new ones. Values not provided are not updated. Currently, the only property that is modifiable is name.

      Example Request Body

      {
    "name": "John Jacob Jingleheimer's List"
}

      Example Response Body

      {
    "id": 3,
    "oid": 53,
    "remote_id": 124587,
    "name": "John Jacob Jingleheimer's List",
    "created_at": "2018-02-13 14:11:22",
    "updated_at": "2018-03-01 01:57:03",
    "invalidated_at": null
}

      Delete an Account

      DELETE /emma/v1/account/{id}?oid={oid}

      {id} is the ID of the Account you wish to hard delete. {oid} is required to make sure the proper account is deleted. Only available if the user has the Admin or Console role.

      Example Request

      DELETE /emma/v1/account/3?oid=53

      Update Account Api Keys

      PUT /emma/v1/account/{id}/credentials?oid={oid}

      {id} is the id of the Account that you are modifying. {oid} is required to make sure the proper account is modified.

      Example Request

      PUT /emma/v1/account/1/credentials?oid=148

      Example Request Body

      {
    "public_key": "dGhpcyBpcyBhIHRlc3Qga2V5Cg==",
    "private_key": "opeiglkzj39ask29afk2ksdjgk"   
}

      Update Members in Bulk

      PUT /emma/v1/member/bulk

      Updates the provided members based on their ID values. Cannot update more than 500 members at a time.

      Example Request Body

      [
    {
        "id":1,
        "remote_id" : 11,
        "account_id" : 2,
        "contact_id" : 33,
        "oid":158,
        "name" : "Bob Bobbson",
        "email" : "bob@gmail.com",
        "emma_member_data": {
            "first_name" : "Bob",
            "last_name" : "Bobbson"
        },
        "evertrue_member_data": {
            "first_name" : "Bob",
            "last_name" : "Bobbson"
        },
        "in_emma" : true,
        "should_send" : true
    }
]

      Update Members should_send field

      PUT /emma/v1/member/{id}?should_send=true

      {id} is the member ID that needs to be updated. should_send is the updated value to be passed into the member's should_send field.

      Delete a Member

      DELETE /emma/v1/member/{id}?oid={oid}

      {id} the ID of the member to delete. {oid} the oid of the member to delete. Only available if the user has the Admin or Console role.

      Delete Members in bulk

      DELETE /emma/v1/member?id[]=1&id[]=2&id[]=3

      id[] is a list of IDs of the members to delete. {oid} the oid of the member to delete. Only available if the user has the Admin or Console role.

      Update Group Connection

      POST /emma/v1/group/{id}/connection?et_list_id=274

      {id} is the group ID that you want to update. et_list_id is the new evertrue list value to write to et_list_id.

      Delete Group

      DELETE /emma/v1/group/{id}

      {id} is the ID of the group you want to delete. Only available if the user has the Admin or Console role.

      Delete Groups in Bulk

      DELETE /emma/v1/group/bulk?id=[1,2]

      id[] is the array of group IDs you wish to delete. Only available if the user has the Admin or Console role.

      List API

      Push a List to Emma

      POST /emma/v1/list/push?list_id=4240&account_id=8

      list_id is the ID for the list. See https://github.com/evertrue/contacts#lists for more on how to make lists. account_id is the ID of the account you want to push this list to. (This is NOT the remote ID) If the group for this list does not exist, it is created. Same with Members and Memberships. Everything is then pushed to MyEmma through their API. You must be the list owner to push the list to your account.

      Get List Info

      GET /emma/v1/list/{id}?account_id=10003

      {id} is the EverTrue list ID you want to return data on. account_id is an optional value for the internal EverTrue account ID. When account_id is provided, the info for the list is limited to info from only that account.
